javascript中构造函数继承关系的整理

发表于 2019-12-01 更新于 2021-04-16 分类于 前端学习

一、传统模式（原型链）

Grandfather.prototype = { name:"hong"};
function Grandfather(){}
var grandfather = new Grandfather();
Father.prototype = grandfather;
function Father(){}
var father = new Father();
son.prototype = father;
function Son(){}
var son = new Son();

缺点：会导致不需要的方法也被子类继承

二、借用构造函数

function Person(name,age,sex){
    this.name = name;
    this.age = age;
    this.sex = sex;
}
function Student(name,age,sex,grand){
    Person.call(this,name,age,sex);
    this.grand = grand;
}

缺点：会多调用函数影响执行速度

三、共享原型

Father.prototype = { name:"hong"}
function Father(){}
var father = new Father();
Son.prototype = Father.prototype;
function Son(){}
var son = new Son();

缺点：修改子类的原型会导致父类原型发生变化

四、圣杯模式

function inherit(Target,Origin){
    function F(){};
    F.prototype = Origin.prototype;
    Target.prototype = new F();
    Target.prototype.constuctor = Target;
    Target.prototype.uber = Origin.prototype;
}

直接使用这个函数，从Origin继承来的Target可以消除前三种模式的缺点
